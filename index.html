Bopomofo Character Recognition Model

Tech: Python, Machine Learning, TensorFlow, OpenCV, Pillow, CNN, Jupyter Notebook

Full Jupyter Notebook Project Repository

Overview

This project aims to build a machine-learning model that can recognize individual Bopomofo symbols. Bopomofo is a phonetic character system used in Taiwan, with limited popularity and research in the machine learning domain. To enhance the model's usefulness, I trained it on both computer fonts and handwritten characters, requiring a well-structured dataset and robust training methods to prevent overfitting.

To achieve accurate recognition, I used a Convolutional Neural Network (CNN), a type of deep learning model specifically designed for image processing. CNNs excel at recognizing patterns in images by using multiple layers to detect edges, shapes, and complex features, making them well-suited for distinguishing Bopomofo characters despite variations in handwriting and font styles.

Despite the constraints of a small and imbalanced datasetâ€”13 computer fonts with varying weights and two handwritten samplesâ€”the model achieved impressive results:

  • Computer fonts: 85-100% accuracy
  • Handwritten characters: 75-85% accuracy

This highlights the effectiveness of advanced data augmentation techniques and balancing strategies. The project demonstrates the feasibility and my ability of achieving high performance in machine learning tasks with limited resources.

Project Details

Data Collection & Preparation

  • Gathered a custom dataset by collecting handwriting samples from friends and family.
  • Collected over a dozen open-source computer fonts to improve generalization.
  • Used the Python library Pillow to generate individual character images from font files using the ImageFont feature.

First Attempt: Early Training Challenges

Initially created training images manually by taking screenshots and splitting them into individual character images.

Issue:

  • The splitting process caused misalignment, leading to position-dependent bias in training.
  • Model performed better on computer fonts (84%) than handwritten samples (65%), suggesting the need for data augmentation to compensate for the smaller sample size of handwritten fonts.
  • Noticed that character order in the training set influenced model accuracyâ€”matching order improved accuracy by 10%.

Second Attempt: Improved Training Process

  • Automate dataset creation by generating individual character images from font files. Speeding up preprocessing and avoiding aliment issue in manual screenshot causing bias.
  • Implemented data augmentation to improve robustness:
    • Randomized image alignment to prevent position memorization.
    • Contrast adjustments, slant modifications, and blurring to simulate handwriting variations.
  • Balanced dataset by categorizing fonts into four types:
    • Even strokes
    • Dynamic strokes
    • Heavily stylized fonts
    • Handwritten samples
  • Used OpenCV for augmentation and image processing.
  • Used Python library Pillow's ImageFont feature to automate dataset creation by generating individual character images from font files. Greatly speeding up preprocessing and avoiding aliment issue in manual screenshot causing bias.

Results & Performance

  • Training epochs reduced from 200 to 30 while improving accuracy.
  • Model performance:
    • Computer fonts: 85-100% accuracy
    • Handwritten characters: 75-85% accuracy
    • More off-centered images still lower accuracy but with less bias.
  • Additional insights:
    • Despite a small dataset (13 fonts and two handwritten samples), the model achieved high accuracy rates.
    • Data augmentation techniques significantly improved model robustness against variations in handwriting and font styles.
    • Evaluated on entirely new fonts and handwriting samples, the model showcased excellent generalization, validating its potential for real-world applications.

What I Learned

  • Used Python Pillow library's ImageFont feature to efficiently generate images of individual characters from font files, significantly improving dataset creation speed and reduce bias.
  • Leveraged OpenCV for advanced data augmentation techniques to enhance model robustness.
  • Importance of balanced and diverse training data for handwritten recognition.
  • Data augmentation significantly improves model robustness.
  • Structured folder organization and automated testing setup is crucial to experiment with different methods and quickly iterate.
  • Implemented logging with JSON files and automated modal evaluation visualization is crucial to experiment with different methods and quickly iterate.

AI-Based Text Recognition
